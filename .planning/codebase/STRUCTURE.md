## Directory Purposes

**public/:**
- Purpose: Static files served directly by the web server
- Contains: HTML template, favicon, PWA manifest, PNG logos
- Key files: `public/index.html` - the single HTML file that React mounts into

**src/:**
- Purpose: All application source code and styling
- Contains: React components, business logic, styles, tests
- Key files: `src/App.js` (root), `src/helpers.js` (calculations)

**src/components/:**
- Purpose: Presentational React components that render UI
- Contains: Functional components for specific UI sections
- Key files:
  - `EmployeeDetails.js` - Employee classification, type, age, custom rate inputs
  - `WorkHours.js` - Weekly schedule table and calculate button
  - `Allowances.js` - Allowance checkboxes and numeric inputs
  - `PaySummary.js` - Summary view of calculated pay
  - `DetailedBreakdown.js` - Detailed tables showing day-by-day and segment breakdowns

**.planning/codebase/:**
- Purpose: Machine-readable codebase documentation for orchestrator tools
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, STACK.md, CONCERNS.md
- Generated by: GSD mapping tools
- Not committed to git

## Naming Conventions

**Files:**
- React components: PascalCase.js (e.g., `EmployeeDetails.js`, `PaySummary.js`)
- Utilities/helpers: camelCase.js (e.g., `helpers.js`)
- Styles: matching component name or `.css` (e.g., `App.css`, `index.css`)
- Tests: [ComponentName].test.js (e.g., `App.test.js`)

**Directories:**
- Feature folders: lowercase plural (e.g., `components/`)
- Utility folders: lowercase descriptive (e.g., `.planning/`)

**React Components:**
- Component names: PascalCase, match filename exactly
- Props: camelCase (classification, setClassification, handleTimeChange)
- State variables: camelCase with semantic names (weeklyData, customRate, results)
- Event handlers: camelCase starting with "handle" (handleTimeChange, handlePublicHolidayChange)

**Constants:**
- Data lookups: camelCase (pharmacyAwardRates, classifications, weekDays)
- CSS classes: kebab-case (app-header, section-header, breakdown)

**Functions:**
- Calculation functions: camelCase, action-oriented (calculatePayForTimePeriod, calculateBreakTime)
- Getters: getPenaltyRateDetails, getPenaltyDescription
- Handlers: handleXxx pattern
- Pure functions: camelCase (parseFloat, format, differenceInMinutes via date-fns)

## Where to Add New Code

**New Feature - Payment Calculation Enhancement:**
- Primary code: `src/helpers.js` - Add new calculation function or extend existing penalty logic
- UI: `src/components/` - Create new component or extend existing form component
- State: `src/App.js` - Add new state variable with useState, update handler functions
- Example: Adding overtime request tracking would require new state in App.js, new fields in Allowances.js, and new logic in calculatePayForTimePeriod()

**New Component/Module - New Form Section:**
- Implementation: Create `src/components/NewFeatureName.js` following existing component patterns
- Props: Design to accept values from App.js state and callback functions for updates
- Styling: Add CSS classes to `src/App.css` using existing naming patterns (class names tied to section headers)
- Integration: Import in `src/App.js` and place in appropriate grid/layout location
- Example: A new "Leave Entitlements" component would be created as `src/components/LeaveEntitlements.js`

**Utilities/Helpers:**
- Shared calculation logic: `src/helpers.js` - Add exported function or data structure
- Data lookups: Add to existing arrays (classifications, weekDays) or create new export
- Test with: Reference in helpers and update any dependent components

**Tests:**
- Component tests: `src/components/ComponentName.test.js`
- Utility tests: `src/helpers.test.js` (if created)
- Run with: `npm test`

## Special Directories

**node_modules/:**
- Purpose: NPM dependencies
- Generated: Yes (by npm install)
- Committed: No (.gitignore excludes)
- Contents: React, ReactDOM, date-fns, testing libraries, build tools

**build/:**
- Purpose: Production-ready bundled code
- Generated: Yes (by npm run build)
- Committed: No (.gitignore excludes)
- Contents: Minified JS/CSS, index.html, optimized assets

**.git/:**
- Purpose: Version control metadata
- Generated: Yes (by git init)
- Committed: N/A (Git system directory)
- Excludes: Secrets, build artifacts, node_modules per .gitignore

**.vscode/:**
- Purpose: VS Code editor configuration
- Generated: No (manually created)
- Committed: Selectively (settings.json may be committed for consistency)
- Contents: Editor settings, extensions, debug configuration
