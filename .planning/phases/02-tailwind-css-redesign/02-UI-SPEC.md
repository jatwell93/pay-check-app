| Property | Value |

## Spacing Scale

Declared values (all multiples of 4, per design system standard):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, cell padding |
| md | 16px | Default form input padding, section gaps |
| lg | 24px | Section padding, major layout gaps |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks, modal padding |
| 3xl | 64px | Page-level spacing (reserved, not primary use) |

**Tailwind mapping:** `p-1` (4px) → `p-6` (24px); `gap-4` (16px) → `gap-8` (32px); `mb-8` (32px) for section breaks.

Exceptions: **None declared.** All spacing uses 8-point scale multiples.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 400 (regular) | 1.5 |
| Heading 2 | 20px | 600 (semibold) | 1.2 |
| Heading 3 | 18px | 600 (semibold) | 1.2 |
| Display (Page title) | 28px | 700 (bold) | 1.1 |

**Tailwind mapping:**
- Body: `text-base font-normal leading-relaxed`
- Label: `text-sm font-normal leading-relaxed`
- Heading 2: `text-xl font-semibold leading-tight`
- Heading 3: `text-lg font-semibold leading-tight`
- Display: `text-3xl font-bold leading-tight`

**Font families:** System font stack (no custom typeface). Font is applied globally in `src/index.css`.

**Constraint:** Two font weights max (400 regular, 600 semibold, 700 bold for headings only). Do NOT use 500 or other weights.

## Layout Contract (Locked from CONTEXT.md)

### Header
- **Location:** Top of page, full-width
- **Styling:** `bg-slate-900 text-white py-4 shadow-md`
- **Container:** `max-w-4xl mx-auto px-4` (centered max-width container)
- **Title:** `<h1 className="text-3xl font-bold">Pay Check App</h1>`
- **Subtitle:** `<p className="text-gray-300 text-sm mt-1">Check if you're being paid correctly</p>`
- **Constraint:** Static subtitle; does NOT change with selected award (CONTEXT.md D-02)

### Error Banner (Locked from CONTEXT.md D-09, D-10)
- **Location:** Below header, above AwardSelector
- **Styling:** `bg-red-50 border-l-4 border-red-600 p-4 rounded`
- **Close button:** `× (text-red-700 hover:text-red-900, aria-label="Dismiss error")`
- **Visibility:** Only shown when `awardError` state is non-null
- **Dismissal:** Clicking × sets `awardError` to null

### Loading Overlay (Locked from CONTEXT.md D-08)
- **Location:** Covers form area below header (not entire viewport)
- **Styling:** `fixed inset-0 bg-black/50 z-40 flex items-center justify-center`
- **Content:** White card with spinner (`w-10 h-10 border-4 border-gray-200 border-t-blue-600 animate-spin`) + text "Loading award rates..."
- **Visibility:** Only shown when `awardLoading` is true
- **Accessibility:** Form is NOT accessible during load (overlay blocks interaction)

### Grid Layout (Locked from CONTEXT.md D-05)
- **Pattern:** `grid grid-cols-1 md:grid-cols-3 gap-6 mb-8`
- **Behavior:** 1 column on mobile, 3 columns on medium screens (≥768px)
- **Components:** EmployeeDetails (2 cols) + Allowances (1 col) on desktop, stacked on mobile

### WorkHours Table (Locked from CONTEXT.md D-06)
- **Pattern:** Wrapped in `<div className="overflow-x-auto">`
- **Behavior:** Horizontal scroll on small screens (<500px); table width maintains at 100%
- **Styling:** `border-collapse`, `w-full`

### Section Order (Locked from CONTEXT.md D-07)
1. Header
2. Error Banner (if awardError)
3. Loading Overlay (if awardLoading)
4. AwardSelector
5. EmployeeDetails + Allowances (3-col grid)
6. WorkHours
7. OverviewBreakdown
8. ImportantNotes
9. Footer (tidy styling, styling discretionary)

### Weekly Summary Row (Locked from CONTEXT.md D-12, D-13)
- **Location:** Below OverviewBreakdown table
- **Content:** "Calculated: ${total}  \| Paid: ${actual}  \| Difference: ${diff}"
- **Styling:** Prominent row with status badge (green/red/yellow)
- **Visibility:** Completely hidden until at least one "actual paid" amount is entered (no placeholder)
- **Logic:** Check `actualPaidByDay.some(x => x !== '' && x !== null && !isNaN(parseFloat(x)))` to show row

## Form Styling Contract

**Input fields (all text, number, time inputs):**
```
className="w-full p-3 border border-gray-300 rounded-md focus:ring-2 focus:ring-blue-500 focus:border-transparent placeholder-gray-400 text-gray-700"
```

**Breakdown:**
- `w-full` — full width of parent container
- `p-3` — 12px padding all sides
- `border border-gray-300` — 1px solid light grey border
- `rounded-md` — 6px border radius
- `focus:ring-2 focus:ring-blue-500` — 2px blue ring on focus (WCAG 2.4.7 compliance)
- `focus:border-transparent` — hide border during focus (ring takes precedence)
- `placeholder-gray-400` — light grey placeholder text
- `text-gray-700` — input text colour

**Select dropdowns:**
```
className="w-full p-3 border border-gray-300 rounded-md focus:ring-2 focus:ring-blue-500 focus:border-transparent text-gray-700 bg-white"
```

**Checkboxes (Allowances):**
- Style with Tailwind: `w-4 h-4 accent-blue-600 cursor-pointer`
- Label adjacent: `<label className="inline-flex items-center gap-2"><input type="checkbox" /> <span>Allowance Name</span></label>`

**Buttons:**

| Button Type | Styling |
|------------|---------|
| Primary CTA ("Calculate Pay") | `bg-emerald-500 text-white font-medium py-2 px-4 rounded-md hover:bg-emerald-600 disabled:bg-gray-400 disabled:cursor-not-allowed` |
| Danger/destructive | `bg-red-600 text-white py-2 px-4 rounded-md hover:bg-red-700` (not used in Phase 2) |
| Secondary/close button (× on error) | `text-red-700 hover:text-red-900 hover:bg-red-100 rounded p-1 font-bold text-lg leading-none` |

## Responsive Design Contract

**Breakpoints (Tailwind default):**
- Mobile first (no prefix): <640px
- `sm:` → 640px
- `md:` → 768px (primary breakpoint for this phase)
- `lg:` → 1024px
- `xl:` → 1280px

**Mobile testing requirement:**
- Test at 375px (iPhone SE) — forms should be usable, table should scroll horizontally
- Test at 768px (iPad) — 3-col grid should display, all sections readable
- Test at 1024px (desktop) — full layout visible without scrolling (if viewport permits)

**Constraint:** All interactive elements (buttons, inputs) must have touch targets ≥44px (standard). Tailwind padding (`p-3` = 12px) + font size (14–16px) should meet this.

---

## Accessibility Contract

**Focus indicators:**
- All inputs, buttons, and clickable elements must have visible focus ring (`focus:ring-2 focus:ring-blue-500`)
- Focus order: header → error banner → award selector → form sections (top to bottom) → calculate button

**Color contrast:**
- Text on white background: navy `#1e3a5f` (AAA compliance ✓)
- Text on navy header: white (AAA compliance ✓)
- Status badges (green/red/yellow): verified in OverviewBreakdown.js (AAA compliance ✓)
- Placeholder text (gray-400): check contrast ratio; may need fallback to gray-500 if <4.5:1

**ARIA labels:**
- Error banner close button: `aria-label="Dismiss error"`
- Loading overlay: no ARIA needed (form is blocked, overlay is not interactive)
- Status badges: semantic meaning in text ("Underpaid") is sufficient; no ARIA required

**Keyboard navigation:**
- All form inputs must be reachable via Tab key
- Calculate button must be activatable via Enter key
- Error banner close button must be activatable via Space or Enter key

## Tailwind Configuration

**File:** `tailwind.config.js` (generated by `npx tailwindcss init`)

**Required content paths:**
```js
content: [
  "./src/**/*.{js,jsx,ts,tsx}",
```

## Implementation Notes

**Key dependencies to install:**
```bash
npm install -D tailwindcss postcss autoprefixer
npx tailwindcss init
```

**Files to modify:**
- `src/index.css` — add `@tailwind` directives (replace existing global styles)
- `src/App.js` — replace `.container`, `.app-header` CSS classes with Tailwind utilities
- `src/App.css` — remove or archive (classes will be replaced by Tailwind)
- All component files in `src/components/` — convert CSS classes to Tailwind utilities

**Files to preserve:**
- `src/helpers.js` — calculation logic unchanged
- `src/config/awardConfig.js` — data unchanged
- `src/services/awardRatesService.js` — API logic unchanged
- All test files — existing tests verify behaviour, not CSS; no test changes required

**PostCSS configuration:**
- Automatic: `npx tailwindcss init` creates `tailwind.config.js` and updates `postcss.config.js`
- No manual PostCSS edits needed for standard CRA setup

**Browser support:**
- Tailwind v3 supports all modern browsers (Chrome, Firefox, Safari, Edge)
- No IE11 support required (CRA 5.0.1 targets modern browsers only)
